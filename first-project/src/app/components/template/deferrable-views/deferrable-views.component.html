Deferrable views

In this section we will discuss how to use deferred views. Deferred views are views that do not get executed immediately when they are created, but instead wait until a later time.

Options available for defer conditional's on:

  • idle - loaded immediatelly after the browser finish to load everything else
  • timer - loaded after a give time
  • viewport - loaded after the defer component is displayed
  • interaction - loaded after the interaction with another content on screen
  • hover - loaded after the mouse hover event
  • immediate - loaded immediatelly, even if the browser didn't finish loading everything else
  • prefetch - loaded after a condition, and only display the content after the first condition is match

The when condition doesn't have options, and it only works with a boolean value after it.

@defer(when isTrue) { } @placeholder {

Retrieving data

} @loading {

Loading...

} @error {

Error!

}
